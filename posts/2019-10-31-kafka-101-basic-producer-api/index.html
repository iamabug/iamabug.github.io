[kafka-101-2] 使用Java API生产数据

 2019.10.31  2020.11.11  Kafka 101  2068  5 mins  

上篇文章里，我们学习了在命令行生产和消费数据，但是在实际的业务场景中，往往要直接通过代码来操作，命令行更多的是用于功能和数据验证，在这篇文章里我们学习如何用Java API来生产数据。

// 创建ProducerRecord对象，一个ProducerRecord对应一条消息，topic为test，key为iamabug，value为大数据学徒，发送前先确认topic存在
// value为实际发送的数据，关于key的含义和作用到到下一篇文章学习
ProducerRecord<String, String> record = new ProducerRecord<String, String>("test", "iamabug", "大数据学徒");
// send方法会抛出异常，这里省略了try/catch
producer.send(record);

上面的代码是基本流程，实际上有三种方式来发送数据，分别为：

  1. Fire-and-Forget：只管发，不关心Kafka是否接收到数据，大部分情况消息都可以正常到达，且Producer内置了自动重试机制，但这种方式仍有可能丢失数据；
  2. 同步发送：发送数据并等待结果，等待过程是阻塞的；
  3. 异步发送：发送的时候提供一个回调函数，当结果返回时，由回调函数结果进行相应的处理；

public class BasicProducer {

    private static KafkaProducer<String, String> producer;

    public static void main(String[] args) {
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        producer = new KafkaProducer<String, String>(props);
        ProducerRecord<String, String> record = new ProducerRecord<String, String>("test", "iamabug", "大数据学徒");

      	// 根据需要，注释下面的函数调用
        fireAndForget(record);
      	synchronousSend(record);
      	asynchronousSend(record);
    }
		
  	// Fire-and-Forget
    private static void fireAndForget(ProducerRecord record) {
    }
  	
  	// 同步发送
    private static void synchronousSend(ProducerRecord record) {
    }
  
  	// 异步发送
    private static void asynchronousSend(ProducerRecord record) {
    }
}

Fire-and-Forget

先将 fireAndForget 的函数体写为：

1
2
3
4
5

try {
    producer.send(record);
} catch (Exception e) {
    e.printStackTrace();
}

运行代码，会发现 kafka-console-consumer 的命令行竟然没有输出，这是怎么回事？原因是，**执行 producer.send(record) 会将数据交给叫做 Sender 的线程，由 Sender 线程负责发送数据，当前的主线程就继续执行后面的代码，但我们并没有后续的代码，并且 Sender 线程是特殊的 Daemon，所以当主线程退出时，整个进程就结束了，因此数据还没有来得及发送。**这种现象在实际的业务中不太可能出现，不过可以通过如下的方式来验证这种解释是否正确（注意：省略了try/catch语句）：

1
2
3

producer.send(record);
// 主线程睡眠10 ms，等待Sender线程完成发送，如果10 ms不够，可以适当增大
Thread.sleep(10);

这时应该可以从 kafka-console-consumer 的命令行观察到输出了，但是需要注意，如果发送失败，比如网络不通，我们是感知不到的，我推测 Fire-and-Forget 这种方式应该很少使用。

// get方法还有一个重载方法，可以传入阻塞时间作为参数，超时则抛出异常
RecordMetadata result = (RecordMetadata) producer.send(record).get();
System.out.println("Topic: " + result.topic());
System.out.println("Partition: " + result.partition());
System.out.println("Offset: " + result.offset());
System.out.println("Timestamp: " + result.timestamp());

同步发送成功后可以得到这条消息的相关信息，比如这条消息写入了哪个Topic的哪个Partition，它的offset是多少，时间戳也都可以获得。

同步发送的缺点也很明显：在发送消息时，程序是阻塞的，效率低。

import org.apache.kafka.clients.producer.Callback;

private void asynchronousSend(ProducerRecord record) {
  	// 比前两种方法多了一个回调函数的参数
    // 不需要try/catch，因为异常会在回调函数中处理
    producer.send(record, new MyProducerCallback());
  	// 和Fire-and-Forget情形一样，需要保证主线程不能再执行send方法后立即退出
 		try {
        Thread.sleep(10);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
}

private class MyProducerCallback implements Callback {
    // 这个接口需要实现onCompletion方法
    public void onCompletion(RecordMetadata metadata, Exception exception) {
        if (null != exception) {
          // 消息发送失败的处理方式，实际生产中可以继续重试，或者在日志中记录错误
           exception.printStackTrace();
        } else {
            System.out.println("Offset: " + metadata.offset());
        }
    }
}

很显然，使用回调函数的异步发送的方式既高效，又优雅，这种方式在实际应用中会很广泛。

总结

在这篇文章中我们学习了如何用三种不同的方式来向Kafka生产数据：

  1. Fire-And-Forget：将数据交给 Sender 线程就不管了，不可靠；
  2. 同步发送：阻塞等待结果，可以得到每条消息的结果，但是低效；
  3. 异步发送：通过回调函数实现，可靠又高效。
